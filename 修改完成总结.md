## 📊 技术架构

### 请求流程
```
HttpApi 接收 POST /api/v1/Capacity/hourly
    ↓
ReceiveHourlyCapacityCommand Handler
    ↓ 验证设备
    ↓
发布 HourlyCapacityReceivedEvent 到 MQ (RabbitMQ)
    ↓
DataWorker 消费 HourlyCapacityReceivedEvent
    ↓
HourlyCapacityConsumer 转发到 UpsertHourlyCapacityCommand
    ↓
MediatR Pipeline + DistributedLockBehavior
    ↓
UpsertHourlyCapacityHandler (串行执行，并发安全)
    ↓ Upsert 数据库
    ↓ 清除缓存
```

### 分布式锁
- **锁键格式**：`iiot:lock:capacity:hourly:{DeviceId}:{Date}:{Hour}:{Minute}:{ShiftCode}`
- **超时时间**：5 秒
- **并发限制**：同一槽位同班次只有 1 个 Consumer 处理
- **实现**：Redis 分布式锁（通过 `DistributedLockBehavior`）

### 缓存策略
- **查询缓存**：5 分钟 TTL
- **缓存清除**：Upsert 后立即清除对应的缓存键

## 📝 关键代码要点

### 分布式锁属性
```csharp
[DistributedLock("iiot:lock:capacity:hourly:{DeviceId}:{Date}:{Hour}:{Minute}:{ShiftCode}", TimeoutSeconds = 5)]
public record UpsertHourlyCapacityCommand(...) : IRequest;
```

### Upsert 实现
```csharp
var existing = await dbContext.HourlyCapacities
    .FirstOrDefaultAsync(c => c.DeviceId == request.DeviceId
                           && c.Date == request.Date
                           && c.Hour == request.Hour
                           && c.Minute == request.Minute
                           && c.ShiftCode == request.ShiftCode,
        cancellationToken);

if (existing is not null)
{
    existing.UpdateCapacity(request.TotalCount, request.OkCount, request.NgCount);
}
else
{
    dbContext.HourlyCapacities.Add(new HourlyCapacity(...));
}
```

### SQL 聚合维度
```sql
SELECT date, hour, minute, time_label, shift_code, 
       total_count, ok_count, ng_count,
       ROUND(ok_count * 100.0 / NULLIF(total_count, 0), 2) AS ok_rate
FROM hourly_capacity
ORDER BY date DESC, hour DESC, minute DESC
```
