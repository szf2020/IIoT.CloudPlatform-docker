# EF Core 迁移命令备忘录 (.NET Aspire 环境专用)

> **核心思想**：在 .NET Aspire 架构下，严禁在代码中硬编码连接字符串。所有 EF Core CLI 操作均通过**终端临时注入环境变量** (`ConnectionStrings__<资源名>`) 的方式执行，保证代码的绝对纯净。

## ⚙️ 占位符说明 (执行前请替换)

- **`[DB_NAME]`**: AppHost 中注册的数据库资源名称（例如 `iioTdb`）。
- **`[CONN_STR]`**: 真实的物理库连接字符串（例如 `Host=localhost;Database=IIoT;Username=postgres;Password=123`）。
- **`[EF_PROJ]`**: 包含 `DbContext` 的类库项目路径（例如 `src/infrastructure/IIoT.EntityFrameworkCore`）。
- **`[API_PROJ]`**: 包含 `Program.cs` 的启动项目路径（例如 `src/hosts/IIoT.HttpApi`）。

------

## 1. 删除后重新生成迁移文件

*适用场景：开发初期模型变动剧烈，想要抹平历史记录，重新生成一个干净的基础迁移。*

### 步骤 1.1：清理物理文件与旧库

1. 手动删除 `[EF_PROJ]` 项目下的 `Migrations` 文件夹。
2. （可选但推荐）使用 Navicat/pgAdmin 等工具，直接将本地开发库 Drop 掉并重新建一个空库，防止旧表与新迁移冲突。

### 步骤 1.2：生成全新的迁移文件 (命名为 `InitialCreate`)

**💻 PowerShell (Windows 推荐):**

PowerShell

```
$env:ConnectionStrings__[DB_NAME]="[CONN_STR]"
dotnet ef migrations add InitialCreate --project [EF_PROJ] --startup-project [API_PROJ]
```

**🐧 Bash / Zsh (Mac/Linux):**

Bash

```
ConnectionStrings__[DB_NAME]="[CONN_STR]" dotnet ef migrations add InitialCreate --project [EF_PROJ] --startup-project [API_PROJ]
```

------

## 2. 更新迁移文件 (同步到数据库)

*适用场景：刚执行完 `add` 命令生成了新的迁移文件，现在需要将结构正式应用到物理数据库中。*

**💻 PowerShell (Windows 推荐):**

PowerShell

```
$env:ConnectionStrings__[DB_NAME]="[CONN_STR]"
dotnet ef database update --project [EF_PROJ] --startup-project [API_PROJ]
```

**🐧 Bash / Zsh (Mac/Linux):**

Bash

```
ConnectionStrings__[DB_NAME]="[CONN_STR]" dotnet ef database update --project [EF_PROJ] --startup-project [API_PROJ]
```

------

## 🛠️ 附加高频操作：追加常规迁移

*适用场景：在已有库的基础上新增了字段或表，需要追加迁移记录。*

**💻 PowerShell 示例:**

PowerShell

```
# 1. 追加新迁移 (例如命名为 AddDeviceMac)
$env:ConnectionStrings__[DB_NAME]="[CONN_STR]"
dotnet ef migrations add AddDeviceMac --project [EF_PROJ] --startup-project [API_PROJ]

# 2. 更新到数据库
$env:ConnectionStrings__[DB_NAME]="[CONN_STR]"
dotnet ef database update --project [EF_PROJ] --startup-project [API_PROJ]
```

> **💡 架构师提示**：
>
> 终端里通过 `$` 或前置赋值注入的环境变量只在当前会话窗口有效。只要关闭终端，配置就会自动失效，绝对不会污染您的宿主机环境变量。
